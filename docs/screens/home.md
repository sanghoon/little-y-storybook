# 홈 화면 스펙

## 1) 목적 / 핵심 행동
- 사용자가 빠르게 이야기를 찾고 읽기를 시작하도록 유도
- 최근에 읽던 회차로 이어보기 제공

## 2) 주요 UI 구성요소
### 상단 영역
- 로고/서비스명
- 검색 진입 버튼(아이콘)

### 이어보기 섹션
- 최근 N개(기본 5) 카드 목록
- 카드 정보: 제목, 연령, 길이, 마지막으로 연 회차 표시

### 큐레이션 섹션
- 추천 섹션(초기에는 고정 리스트로 운영 가능)
- 카드 정보: 제목, 연령, 길이, 태그 1~2개

### 빠른 필터
- 연령(3-5, 6-7, 8-9)
- 길이(짧음/보통/김/연작)

## 3) 상태/예외
- 로딩: 스켈레톤 카드
- 이어보기 없음: “최근 읽은 이야기가 없어요” 안내 + 추천 섹션 강조
- 콘텐츠 없음: “준비 중” 안내 문구

## 4) 사용자 동작/전환
- 카드 탭 → 상세 화면
- 이어보기 카드 탭 → 읽기 화면(저장된 회차)
- 필터 탭 → 리스트 화면(필터 적용)
- 검색 탭 → 검색 화면(리스트)

## 5) 데이터 요구사항
- 이어보기: version id, title, age_range, length_type, last_chapter_index, last_read_at
- 큐레이션: version id, title, age_range, length_type, tags

## 6) 모바일 UX 규칙
- 카드 터치 영역 최소 44px
- 필터 칩은 가로 스크롤 가능
- 한 손 조작을 위해 주요 버튼은 하단/중앙 집중
