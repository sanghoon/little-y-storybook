# 요구사항 문서 (Requirements)

## 기능 요구사항
### 탐색/발견
- 홈에서 추천/신규/인기 이야기 노출
- 태그 필터: 대상 연령, 길이, 테마, 분위기
- 검색: 제목/키워드 기준

### 이야기 상세
- 이야기 소개, 대상 연령, 길이, 테마 표시
- 동일 이야기의 다른 버전 목록/전환
- 시리즈(연작)일 경우 다음 편 안내

### 읽기 화면
- 방해 없는 읽기 모드 (상단 최소 UI)
- 글자 크기 조절
- 페이지/스크롤 진행 표시
- 최근에 읽던 위치 이어보기(로컬 저장)

### 콘텐츠 관리(운영)
- 이야기 기본 정보(Story)와 버전(Version) 분리
- 버전별 태그(연령/길이/난이도) 관리
- 게시/비게시 상태 관리
- 정적 운영: Version 1개 = 파일 1개 (Markdown + YAML frontmatter)

## 비기능 요구사항
- 모바일 브라우저 우선(세로 화면, 엄지 조작)
- 첫 화면 로딩 2~3초 내
- WCAG AA 수준의 가독성(명도 대비, 폰트 크기)
- SEO 기본 메타(타이틀, 설명, OG)

## TTS 확장 가이드 (초안)
- 방식: 브라우저 내장 TTS → 서버 TTS로 확장 가능
- 데이터: audio_url 필드를 optional로 제공(버전/회차)
- fallback: audio_url 없을 때는 텍스트만 표시

## 태그 정책 (초안)
- 대상 연령(만 나이): 3-5, 6-7, 8-9 (구간은 조정 가능)
- 길이: 짧음(3~5분), 보통(6~10분), 김(11~20분), 연작
- tags(선택): 우정, 용기, 가족, 상상, 자연, 동물, 성장, 배려, 호기심, 모험, 잔잔함, 유쾌함, 감동, 교훈,
  창작, 각색/재해석, 고전 원전, 전래/민담, 신화/전설, 세계 명작, 실화 기반

## 이어보기 정책 (초안)
- 저장 위치: 로컬 저장(브라우저 저장소)
- 범위: Story/Version 단위
- 저장 값: 마지막으로 연 회차 index, 마지막 읽은 날짜
- 표기 개수: 최근 N개 (기본 5)
- 저장 시점: 회차 열기 시점에 기록
- 이어보기 기준: 마지막으로 연 회차 index로 복귀
- 정렬: 마지막 읽은 날짜 내림차순
- 표기 개수: 최근 N개 (기본 5)

## MVP 범위 정리
### Must
- 태그 기반 탐색/필터
- 이야기/버전 구조와 전환
- 읽기 모드

### Should
- 검색 자동완성
- 즐겨찾기(로컬 저장)
- 이어보기 히스토리(최근 N개)

### Could
- 읽기 시간 추정치 표시
- 시리즈 연속 읽기 모드

### Won't (초기)
- 회원가입/프로필
- 추천 알고리즘

## 스펙 작성 템플릿
기능별 문서는 아래 포맷을 사용해 기록:
- 목적(Why)
- 사용자 시나리오(Who/When)
- 상세 동작(How)
- 예외/엣지 케이스
- 데이터/태그 영향
- 측정 지표
